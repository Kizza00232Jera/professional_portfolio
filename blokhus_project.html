Blokhus Sports and Events

Product: Responsive dynamic webpage

My role: User research, Web Design, Front-end developer

View website
View Code

Problem and the task

There is no place where you can see all the information for different sports and events happening in Blokhus. Only way to get the information was throught newspaper.

The project group decided to use a Content Management System (CMS) - WordPress and create dynamic website where people could see upcoming and past events.

We started prototyping in multiple iterations moving progressively rom low-fidelity prototypes(sketches) to high-fidelity prototypes. Visual hierarchy and the overall layout was tested using 5-second test. We used Figma as a development tool because it provides shared artboards where it can be collaborated. For building the product, atomic design was used to structure the layout and elements in the page.

Low Fidelity prototype

High Fidelity prototype

HTML, CSS and JavaScript are used, as well as headless CMS (WordPress) for building structure of articles. There was focus on the on-site and off-site SEO which we tried to improve by creating content with relevant keywords, availability to update or add more content(to give visitors reason to keep coming back), having content with proper headings, optimized images, font sizes and typography, and making page responsive for different screen sizes and devices with meta tags included on every page. At the end, we validated the code, tested its readability and page reloading speed.
